Hugo

Node Abhängigkeiten aus Themes

Veröffentlicht am Samstag, 13. März 2021

Mit dem Upgrade auf eine Hugo version auf über 0.75.0 besteht die Möglichkeit das Kommando [hugo mod npm pack](https://gohugo.io/commands/hugo_mod_npm_pack/) zu verwenden um die Node Abhängigkeiten eines Themes mit denen eines Projektes zusammenzuführen.

Dafür müssen die Abhängigkeiten eines Themes in einer Datei package.hugo.json statt package.json gesichert sein, es reicht einfach die von npm oder yarn angelegte Datei umzubenennen. Ein Symlink alleine funktioniert nicht, da Hugo’s virtuelles Dateisystem Symlinks nicht mag.

Hugo Updates

Veröffentlicht am Dienstag, 8. Dezember 2020

Update: Inzwischen wird Hugo 0.82.0 eingesetzt

Bis zum Zeitpunk als dieser Beitrag geschrieben wurde, wurde Hugo 0.74.3 vom 23.7.20 verwendet, damals war Version 0.79 aktuell. Änderungen an der Handling von JavaScript Build-Prozessen verhindern ein Update. Erschwerend kommt hinzu, dass homebrew in letzter Zeit eher verschlimmbessert wurde. Downgrades sind nicht mehr einfach über die Kommandozeile möglich. Inzwischen wird Hugo aber direkt aus den Quellen gebaut, siehe unten.
